# Sales Data Pipeline вЂ” Medallion Architecture

A scalable Azure-based Data Engineering project implementing the **Medallion Architecture (Bronze в†’ Silver в†’ Gold)** using **Azure Data Factory, Azure Databricks, PySpark, Delta Lake, and ADLS Gen2** for processing and analyzing sales datasets. 

---

# Project Overview

This project automates the ingestion, transformation, and analytics preparation of sales-related datasets including:

* Customer
* Product
* Store
* Sales

The pipeline dynamically detects the latest files using Azure Data Factory and processes them through Bronze, Silver, and Gold layers using Databricks and PySpark. 

---

# Architecture

## Medallion Architecture

### Bronze Layer вЂ” Raw Data Ingestion

* Raw CSV files are ingested from ADLS Gen2 using Azure Data Factory.
* Files are copied without transformation.
* Dynamic pipelines identify and process the latest modified files automatically.

Path:

```text
bronze/sales_view/{entity}/
```

---

### Silver Layer вЂ” Data Cleaning & Transformation

* Data cleaning and standardization performed using PySpark.
* Column names converted to `snake_case`.
* Data validation and transformation logic applied.
* Delta tables created for reliable downstream processing.

Path:

```text
silver/sales_view/{table_name}/
```

---

### Gold Layer вЂ” Business Analytics

* Business-ready datasets generated by joining customer, product, store, and sales tables.
* Analytical datasets optimized for reporting and dashboards.

Path:

```text
gold/sales_view/store_product_sales_analysis/
```

# Silver Layer Transformations

## Customer Table

Implemented transformations:

* Converted headers to `snake_case`
* Split `name` into:

  * `first_name`
  * `last_name`
* Extracted domain from email
* Converted gender values:

  * Male в†’ M
  * Female в†’ F
* Split joining date into date and time
* Created `expenditure_status` column

---

## Product Table

Transformations:

* Converted headers to `snake_case`
* Derived `sub_category` using `category_id`

Example:

```text
1 в†’ phone
2 в†’ laptop
3 в†’ playstation
4 в†’ e-device
```

---

## Store Table

Transformations:

* Converted headers to `snake_case`
* Extracted `store_category` from email domain
* Standardized date formats

---

## Sales Table

Transformations:

* Standardized column naming
* Converted dates into `yyyy-MM-dd`
* Stored as Delta table
